# KR VKO

Имитационная модель системы воздушно-космической обороны на Bash. Проект моделирует генерацию целей, работу командного пункта ВКО, радиолокационных станций, зенитных ракетных дивизионов и комплекса СПРО.

Система обнаруживает цели, классифицирует их по скорости, передает события на командный пункт, выполняет пуски ракет/противоракет, фиксирует промахи и уничтожение целей, ведет журналы и сохраняет статистику в SQLite.

## Состав проекта

| Файл | Назначение |
| --- | --- |
| `start.sh` | Запуск всей системы или отдельного компонента |
| `stop.sh` | Остановка всей системы или отдельного компонента |
| `GenTargets.sh` | Генератор целей |
| `kp.sh` | Командный пункт ВКО |
| `rls.sh` | Скрипт РЛС, запускается с номером `1`, `2` или `3` |
| `zrdn.sh` | Скрипт ЗРДН, запускается с номером `1`, `2` или `3` |
| `spro.sh` | Комплекс СПРО |
| `common.sh` | Общие функции: расчеты, обмен сообщениями, HMAC, работа с БД |
| `config.sh` | Конфигурация координат, дальностей, боекомплекта и таймингов |
| `db_queries.sh` | Вывод статистики из базы данных |
| `report.html` | HTML-отчет по проекту |
| `map.png` | Карта/схема для отчета |
| `logs/` | Журналы работы компонентов |
| `db/vko.db` | База данных SQLite |

## Требования

Проект рассчитан на запуск в Linux.

Нужны:

- `bash` версии 4 или выше;
- `sqlite3`;
- `bc`;
- `openssl`;
- `xxd`;
- стандартные утилиты Linux: `find`, `awk`, `sed`, `stat`, `tail`, `wc`.

Для Ubuntu/Debian зависимости можно установить так:

```bash
sudo apt update
sudo apt install -y bash sqlite3 bc openssl xxd
```

Запуск от имени `root` запрещен проверками в скриптах.

## Быстрый запуск

```bash
chmod +x *.sh
./start.sh
```

После запуска создаются рабочие каталоги:

- `logs/` - журналы компонентов;
- `db/` - база данных SQLite;
- `messages/` - файловый обмен сообщениями;
- `temp/` - временные данные и блокировки;
- `pids/` - PID-файлы процессов.

Остановить систему:

```bash
./stop.sh
```

Посмотреть статистику:

```bash
./db_queries.sh
```

## Запуск отдельных компонентов

Можно запускать не всю систему, а отдельные части:

```bash
./start.sh gen
./start.sh kp
./start.sh rls1
./start.sh rls2
./start.sh rls3
./start.sh spro
./start.sh zrdn1
./start.sh zrdn2
./start.sh zrdn3
```

Остановка отдельных компонентов выполняется аналогично:

```bash
./stop.sh rls2
./stop.sh spro
```

### Типы целей

Цель классифицируется по скорости:

| Тип | Обозначение | Скорость |
| --- | --- | --- |
| Баллистический блок | `BB_BR` | от 8000 м/с |
| Крылатая ракета | `KR` | от 250 до 1000 м/с |
| Самолет | `SAM` | от 50 до 249 м/с |

Генератор целей работает в районе `13000000 x 9000000` м и создает до 50 целей одновременно. Вероятность поражения цели при пуске в генераторе задана как 70%.

## Как работает система

1. `GenTargets.sh` создает цели, обновляет их координаты и пишет отметки в `/tmp/GenTargets/Targets`.
2. РЛС сканируют отметки, проверяют попадание в сектор обзора и отправляют на КП данные о баллистических целях.
3. ЗРДН обнаруживают цели типов `KR` и `SAM` в своем радиусе, захватывают цель через механизм блокировки и выполняют пуск.
4. СПРО обнаруживает и поражает баллистические цели `BB_BR`.
5. `kp.sh` принимает сообщения от компонентов, проверяет HMAC, пишет общий журнал и сохраняет события в SQLite.
6. Heartbeat-механизм контролирует доступность РЛС, ЗРДН и СПРО.

## База данных

При запуске создается SQLite-база `db/vko.db`.

Используются таблицы:

- `journal` - общий журнал событий;
- `system_status` - состояния комплексов и остаток боекомплекта;
- `shots` - результаты пусков;
- `nsd_log` - попытки несанкционированного доступа.

Для просмотра статистики:

```bash
./db_queries.sh
```

Скрипт выводит остаток боеприпасов, количество уничтоженных целей, точность систем, последние обнаружения, промахи и события НСД.

## Безопасность обмена

Сообщения между компонентами передаются через файлы в каталогах `messages/to_kp`, `messages/from_kp` и `messages/heartbeat`. Перед отправкой сообщение кодируется в Base64 и подписывается HMAC-SHA256 с ключом из `config.sh`.

Если HMAC не совпадает, КП или компонент фиксирует попытку НСД в журнале и базе данных.

## Отчет

Файл `report.html` содержит готовый отчет по проекту:

- оглавление;
- постановку задачи;
- заданный вариант;
- алгоритмы работы;
- карту с зонами комплексов ВКО;
- отрывки журналов;
- вывод.

Файл можно открыть в браузере или вставить HTML-фрагменты в Moodle.

## Примечания

- На Windows скрипты напрямую не запускаются, используйте Linux или WSL.
- Перед повторным полным запуском `start.sh` очищает старую базу и сообщения.
- Временные цели генератора находятся в `/tmp/GenTargets`.
- Логи ограничиваются параметром `MAX_LOG_LINES` из `config.sh`.
